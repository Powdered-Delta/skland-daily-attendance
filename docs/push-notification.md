# 推送通知变更说明

本文档说明签到任务（`tasks/attendance.ts`）在推送通知与任务结果语义上的改动，便于配置 ServerChan / 其他 statocysts 渠道时对照。

## 变更概览

| 项目 | 变更前 | 变更后 |
|------|--------|--------|
| 部分账号/角色签到失败 | 任务返回 `failed`，GitHub Actions 标红 | 任务返回 `success`，仅推送提示失败 |
| 推送正文 | 含每个角色详情，篇幅很长 | 仅一条短摘要，适配 ServerChan 长度 |
| 游戏维度 | 账号行仅汇总总数 | 按 **舟**（明日方舟）/ **终**（终末地）分别统计 |
| 详细日志 | 与推送混合 | 仅输出到控制台（Actions 日志） |

基础设施类错误（Nitro 启动超时、任务 HTTP 失败等）仍由 `scripts/action.mjs` 标记为工作流失败，行为不变。

## 任务结果语义

- **业务失败**（token 无效、单角色签到失败、授权失败等）：捕获后计入统计并推送，**不**使 `/_nitro/tasks/attendance` 返回 `failed`。
- **执行失败**（服务未启动、请求任务接口失败、未捕获异常）：Actions 退出码非 0。

## 推送格式

- **标题**（不变）：`【森空岛每日签到】`
- **正文**：单行 `\n` 拼接的短文本，由 `buildPushContent()` 生成
- **发送**：仍通过 `statocysts` + `SKLAND_NOTIFICATION_URLS`

### 用语说明

| 文案 | 含义 |
|------|------|
| 新签 N | 本次运行中 newly 签到成功 N 个角色 |
| 已签 N | 该角色今天已签到，本次跳过 N 个 |
| 失败 N | 本次签到失败 N 个 |
| 舟 | 明日方舟（gameId: 1） |
| 终 | 明日方舟：终末地（gameId: 3，多角色独立签到） |

### 正文结构

```
汇总: {成功账号数}成功 {跳过账号数}跳过 {失败账号数}失败 / 共{总账号数}账号
#{index} ✓ {按游戏统计...}          # 账号全部成功
#{index} - 已跳过                   # 该账号今日已在 KV 中标记跳过
#{index} ✗ {错误摘要} {游戏统计} (请更新第{n}个token)  # 失败账号
游戏: 舟 新签a/已签b/失败c | 终 新签d/已签e/失败f   # 全任务按游戏汇总（有角色时）
```

账号级错误信息超过 36 字会截断并加 `…`。

## 示例

### 全部成功

```
汇总: 1成功 0跳过 0失败 / 共1账号
#1 ✓ 舟新签1已签1 终新签2
游戏: 舟 新签1/已签1/失败0 | 终 新签2/已签0/失败0
```

### 部分成功

```
汇总: 1成功 1跳过 1失败 / 共3账号
#1 ✓ 舟新签1 终新签1
#2 ✗ invalid token (请更新第2个token)
#3 - 已跳过
游戏: 舟 新签1/已签0/失败0 | 终 新签1/已签0/失败0
```

### 全部失败

```
汇总: 0成功 0跳过 1失败 / 共1账号
#1 ✗ 处理失败: Request failed… 舟失败1 (请更新第1个token)
游戏: 舟 新签0/已签0/失败1
```

## 实现位置

- 统计与文案：`tasks/attendance.ts`（`AccountResult.byGame`、`buildPushContent` 等）
- 发送通道：`utils/message.ts`（`createMessageCollector` → statocysts）
- 控制台：处理过程使用 `log` / `error`，不再写入推送数组

推送相关纯函数目前保留在 `tasks/attendance.ts` 内；若后续需单测或多渠道模板，可再抽到 `utils/attendance/push-summary.ts`。

## 相关配置

- `SKLAND_NOTIFICATION_URLS`：通知 URL（逗号分隔）
- `SKLAND_TOKENS`：账号 token 列表，失败行中的「第 N 个」与此处顺序一致（从 1 开始）
